# Head First · Design Patterns · C++ Implement

## v1.0
### 总结
- OO基础
抽象、封装、继承、多态
- OO原则
封装变化
多用组合(has-a)，少用继承(is-a)
针对接口编程，不针对实现编程
- OO模式
策略模式——定义算法簇，分别封装起来，让他们之间可以相互替换，此模型
让算法的变化独立于使用算法的客户。
